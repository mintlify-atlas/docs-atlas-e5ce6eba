---
title: "Get Case by ID"
api: "GET /cases/cases/{case_id}"
description: "Retrieve detailed information for a specific medical case including files and AI insights"
---

## Authentication

This endpoint requires authentication via the case_id path parameter.

## Path Parameters

<ParamField path="case_id" type="string" required>
  The unique UUID identifier of the case to retrieve
</ParamField>

## Response

<ResponseField name="case" type="object">
  The complete case object with all details
  
  <ResponseField name="case.case_id" type="string">
    Unique UUID identifier for the case
  </ResponseField>
  
  <ResponseField name="case.user_id" type="string">

  <ResponseField name="case.patient_name" type="string">

  <ResponseField name="case.patient_age" type="integer">

  </ResponseField>
  
  <ResponseField name="case.patient_gender" type="string">
    Patient's gender
  </ResponseField>
  
  <ResponseField name="case.case_summary" type="string">
    Case summary or doctor's notes
  </ResponseField>
  
  <ResponseField name="case.status" type="string">

  <ResponseField name="case.created_at" type="string">

    <ResponseField name="ai_insights.soap_note.assessment" type="string">

## Example Request

```bash
curl -X GET "https://api.medmitra.com/cases/cases/a1b2c3d4-5678-90ab-cdef-1234567890ab" \
  -H "Content-Type: application/json"
```

</ResponseField>

<ResponseField name="400 Bad Request" type="error">

</ResponseField>

  ```json
  }
  ```
</ResponseField>

## Notes

- This endpoint retrieves the most complete information about a case
- AI insights may be null if the background processing has not completed yet
- The `files` array includes all uploaded documents (both lab and radiology)
- Confidence scores range from 0.0 (low confidence) to 1.0 (high confidence)
